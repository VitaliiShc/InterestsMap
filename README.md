# Задача:

Розробити React-компонент, який відображає інтерактивну мапу для пошуку інших користувачів за геолокацією та інтересами.

# Опис:

Уявіть, що на нашій платформі є тисячі користувачів, у кожного з яких вказані геолокація (lat, lon) та список інтересів (теги, наприклад: ["music", "react", "hiking"]). Ваше завдання — створити інтерфейс, де користувачі можуть знаходити однодумців на мапі.

# Ключові вимоги до функціональності:

1. Інтеграція безкоштовної карти: Виберіть та інтегруйте будь-яку безкоштовну бібліотеку для роботи з картами. Популярні варіанти — Leaflet.js (used Leaflet.js) або OpenLayers. Використання платних сервісів, таких як Google Maps API у режимі, що вимагає ключ із прив'язаною платіжною карткою, заборонено.
2. Відображення користувачів: Завантажте мокові дані (~~наданий JSON-файл на 10 000 користувачів~~ (_згенеровано chatGPT_)) і відобразіть кожного користувача на карті у вигляді маркера.
3. Кластеризація маркерів: Оскільки користувачів багато, при віддаленні карти маркери повинні автоматично групуватися в кластери. Кластер має показувати кількість користувачів у ньому. При кліку на кластер карта повинна плавно наближатися, щоб показати згруповані маркери.
4. Картка користувача: При кліку на індивідуальний маркер (не кластер) повинна з'являтися картка (pop-up) з короткою інформацією про користувача: ім'я та список його інтересів.
5. Фільтрація за інтересами: Над картою має бути інпут для фільтрації. Коли користувач вводить у нього інтерес (наприклад, "music"), на карті повинні залишитися тільки ті користувачі (і кластери), у яких є цей інтерес. Фільтрація має працювати швидко, без помітних «гальм».
